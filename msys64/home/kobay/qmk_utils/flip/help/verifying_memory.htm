Verifying Memory

You will usually verify the target memory contents just after a programming operation completes.

Execute Device > Verify

FLIP reads the target memory and compares its contents with the buffer one.

If the Verify operation is successful, FLIP displays the Memory verify pass message in the log message bar; if it fails, FLIP displays the first address which failed.

The Verify operation is performed on the address range (boundaries included) displayed in the Buffer Information frame of the main window.
