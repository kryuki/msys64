Trouble Shooting CAN Interfaces

Cannot open the CAN port

Most of the time, failing to open the CAN port when using FLIP, Batchisp or the ISP Functions Library is due to the fact that the API DLL is missing.

For information about API and drivers installation, read the CAN Pre-Installation Requirements topic.

The IXXAT CANdy dongle does not work

The parallel port should be set up as SPP, Normal or AT in your BIOS settings.

The CANdy dongle has to be connected to the LPT-Port and the power supply before booting the machine.

There is no Windows 2000 driver available for the CANdy dongle.

IXXAT VCI Driver V3

As of version 3.4.0, Flip supports the version 3 of the IXXAT VCI driver. With VCI V3, there is no IXXAT applet available to specify the default hardware.

When running Flip, The IXXAT hardware to be used can be selected from a dialog which pops up when the user clicks the Init button of the CAN Node Setup dialog :

When running batchisp, if only one board is registered, it is selected automatically. If more than one board is registered, the one to be used is selected through an INI file named AtCanIxxat.ini located in the same folder than the AtCanIxxat.dll file.

If the board specified within the INI file cannot be found, or if the INI file does not exist, the first found board is used as fallback.

please read the following instructions.

Before using VECTOR's CAN interfaces to perform ISP with FLIP, you must associate the VECTOR's hardware to this application. Use the VECTOR's applet to do this. The applet may be launched from the Start > Parameters > Configuration Panel menu.

In the following sequence, we will associate the CANpari interface to FLIP.

  1. Launch the VECTOR's applet

  1. Select CANpari channel 1 and click App settings

  1. Click the Add button and type "FLIP" for the Application Name. The Application Settings dialog should now display FLIP:

  1. Click Done.

  1. You may now close the applet and run FLIP.
