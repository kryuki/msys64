Setting Buffer Options

From the top menu bar, execute Buffer > Options

Setting the Buffer Size

By default, the buffer size is device dependent, but you may change it. The new value cannot exceed the device memory size.

Resetting the Buffer

By default, the buffer is not reset when you load a HEX file; this lets you load several HEX files before programming. You may change this option by setting the Yes checkbox.

Setting the Address Programming Range

The address programming range may be controlled in three different ways :

  • the programming range is read from the last loaded HEX file (default option).

  • the address range is the whole buffer one.

  • the programming range is defined by the user.

Whatever the chosen option is, the address programming range is displayed along with its checksum in the Buffer Information frame of the FLIP main window. The checksum is merely the sum of the programming range locations contents.
