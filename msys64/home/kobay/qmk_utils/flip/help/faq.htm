FAQ

Q1 : What are the supported platforms ?

Flip may be used on platforms running Windows 98/Me/NT/2000/XP/Vista and Seven

Linux is supported as well.

Q2 : What about 64-bit platforms ?

The Flip package is released in two versions; one of them houses a 32-bit JRE (Java RunTime Environment). If you are working on a 64-bit machine, you should download and install the 64-bit version of the JRE first, then download the light version of Flip (with no JRE) and install it.

The proper JRE may be downloaded from here : http://java.sun.com/javase/downloads/widget/jdk6.jsp

Q3 : Can I install the USB driver of the Flip package on Windows Vista and Seven 64-bit versions ?

Yes. The USB driver provided by Atmel in the Flip package is now digitally signed and can be installed on Windows Vista and Seven in 64-bit versions.

Q4 : How to configure the AT89C51RE2 device FCB byte ?

Following is an example of a batchisp command file for the AT89C51RE2 device ; it shows how to program the FCB byte.

-device   AT89C51RE2

-hardware RS232

-port COM1

-baudrate 38400

-operation

#----------------- Device Operations -------------------------------

ONFAIL IGNORE

#---------------- Programming sequence ------------------------------

ADDRANGE 0x0000 0x7FFF

FILLBUFFER 0x00

LOADBUFFER "BE2.hex"

ECHO ""

ECHO "Programming the device ..."

ADDRANGE 0x0000 0x7FFF

ERASE F

BLANKCHECK

PROGRAM

VERIFY

#---------------- Device configuration sequence ---------------------

ECHO ""

ECHO "Setting FCB byte (X2 fuse=0, BRV=7)..."

MEMORY CONFIGURATION

ADDRANGE 0x0000 0x0000

FILLBUFFER 0x77

PROGRAM

#---------------- Starting the application code --------------------

ECHO ""

ECHO "Starting application ..."

#START NORESET 0

Q5 : How should I set the security level with batchisp for an AT89C51RE2 ?

I'm trying to set the security level for an AT89C51RE2 device, which works fine with Flip 3.3.4 itself but not with batchisp.

The Flash protection is configured through the HSB byte located at address 0 of Memory Space 7 (see the Datasheet).

Write this HSB as follows :

ECHO " ****************************  "

ECHO " FLASH Security operations "

ECHO " ****************************  "

ECHO "writes Security level 0xFE : read operation not allowed"

MEMORY 7

ADDRANGE 0x0000 0x0000

FILLBUFFER 0xFE

PROGRAM

VERIFY

Q6 : Programming an Atmel device only works the first time.

The first firmware upgrade done with FLIP was OK and my program runs fine, but I am not able to program the part again.

Default factory settings force the device to enter the embedded bootloader program on first power up with BLJB = 0  (checkbox set in Flip GUI) and BSB != 0.

On firmware upgrade completion, Flip sets the BSB (Boot Status Byte) to zero so that the device executes the downloaded firmware on reset. In order to reprogram the part, the user must apply hardware conditions to force the execution of the bootloader.

Please note that both BLJB and BSB settings may be changed manually through the GUI of FLIP to give you total control over the boot process.

Refer to the part data sheet for detailed information about the boot process.
