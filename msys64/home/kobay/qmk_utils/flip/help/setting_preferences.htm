Setting Preferences

From the top menu bar, execute Settings > Preferences.

RS232

The RS232 tab lets you control the boot process hardware conditions if you enable the AutoISP mode by checking the ISP Hardware Conditions Controlled by Flip option.

When the RS232 dialog pops up, Flip builds a list of available COM ports by trying to open them; if a port can be opened, it is added to the list. This process may take a while and you can control the span of the port scan process by setting the Scan COM ports up to text value.

Hex Format

The Hex Format tab lets you specify the HEX output file format (HEX 386 or HEX 86).

Time Out and Logging

The ISP operations performed by Flip are timed-out. In some uncommon circumstances, you may want to increase the time out duration. To do this, simply check the Set New Timeout Value option and enter the time-out value you want to work with.

Check the Log information to flip.log option if you want to send to Atmel a log of the ISP activity in case of trouble using Flip. Please note that checking this option slows down Flip performances.

Connecting and closing

Check the Auto-Connect option if you want to connect automatically to the device the next time you launch Flip.

Check the On Time Out Error option, if you want the communication to be closed when a communication time out occurs.

Check the On Embedded Application Start option, if you want the communication to be closed when the embedded application starts.
